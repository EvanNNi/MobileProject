SecondNest

AI-powered local second-hand marketplace

A nearby exchange network where movers, renters, students and local residents pass on furniture, electronics and daily essentials within walking distance. Camera, GPS, AI recognition, AI pricing and live chat turn scattered posts into a complete local marketplace loop.

Problem & Context

Sellers cannot reach buyers. Buyers cannot find nearby items.

Local second-hand trade is often trapped in social posts and small chat groups. The audience is small, the feed moves quickly, and both sides lack clear price and location signals.

01

Limited selling channels

Listings disappear inside small social groups, so people struggle to sell before moving out.

02

No fair price reference

Sellers know the original price, but not the current local resale value.

03

Nearby discovery is hard

Bulky items are easiest to collect nearby, yet most channels are not location-first.

Core Journey

From photo to listing in under a minute.

Seller and buyer journeys converge around a real photo, a campus-local pickup spot and a live chat.

Seller journey

  1. Photo or album upload
  2. Preview 1-6 item images
  3. AI recognition drafts category, brand and tags
  4. Seller edits details and condition
  5. AI pricing returns a range and suggestion
  6. Publish as a live listing

Buyer journey

  1. Login with a secure account
  2. Browse nearby GPS-ranked listings
  3. Search and filter by price, distance and brand
  4. Inspect product details and map location
  5. Save, like and compare items
  6. Chat with the seller

Application Architecture

Flutter + Firebase + Cloud Functions connect device input to cloud services.

Cloud Functions keep API calls server-side and connect the app to OpenAI and Mapbox. The app is not static: it coordinates local device input with real-time cloud data.

Flutter mobile app

Cupertino UI, camera, gallery, GPS, maps, search, chat and bilingual support.

Firebase

Auth, Google Sign-In, Firestore collections, Storage images and live streams.

Cloud Functions

Secure orchestration for OpenAI recognition, pricing, translation and Mapbox context.

External services

AI Recognition & Pricing

AI drafts the listing. The seller keeps final say.

SecondNest uses a two-stage human-in-the-loop process: first, AI extracts listing metadata from photos; second, AI estimates a price range after seller corrections.

01

Upload item photos

1-6 images go to Firebase Storage and remain linked to the current owner.

02

Recognise the item

OpenAI suggests category, brand, model, condition, description and tags.

03

Estimate price

The seller reviews details first, then AI returns a price range and recommended price.

Data Collection & Handling

User data flows from collection to secure storage.

The app collects only the data needed for a connected marketplace: account identity, profile, GPS location, listing photos, interactions, AI results and chat messages.

Collection

Auth identity, profile, GPS location, item photos, listing interactions and AI outputs.

Processing

AI recognition and pricing, distance sorting, bilingual generation and real-time sync.

Storage

Firestore collections for structured data, Firebase Storage for images and local preferences.

Access & Security

Firebase Security Rules scope writes to the owner. Chat is participant-only. Cloud Functions verify caller identity and image ownership before AI analysis.
